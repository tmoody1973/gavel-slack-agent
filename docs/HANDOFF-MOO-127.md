## Why this exists (the 30-second version)

Local newsrooms are gutted; city hall goes uncovered; accountability dies. Gavel already
pulls the full agenda, summarizes it, detects process anomalies (walk-ons / consent
burials), and knows the sponsors. Point a **journalist lens** at that: "Of the ~200 items
across 8 committees this week, which 5–10 might be a story, and why?" It's dogfoodable for
**Radio Milwaukee** (the user's newsroom — `tarik@radiomilwaukee.org`) and the sharpest
"Agent for Good" angle in the project. The reporter persona already exists (MOO-117).

**Non-negotiable framing (bake into copy + prompts):**
- **Leads, not verdicts** — surface "worth a look" + the record; never assert wrongdoing (defamation/accuracy risk).
- **Explainable** — tags, not a black-box score (mirrors MOO-123's "not ML" rule).
- **Grounded** — the story angle cites the actual matter; never fabricated. Schema-validate it.

## The load-bearing idea: one spine, two lenses

MOO-123 already shipped the reusable selector. **Do not rebuild ranking from scratch.**

```
agent/home/salience.js → selectSalient(upcoming, { boundaries, cap })
  → [{ item, reasons: [{kind:'district'|'walkOn'|'consent'|'big', detail?}] }]
```

- MOO-123 (residents) renders that with a civic-importance lens → "🔎 Discover this week".
- MOO-127 (reporters) **re-scores the same items** with a newsworthiness lens + a Claude
  story-angle → "📰 Story leads". Same selection, different rubric + framing.

## The MOO-127 contract (what to build)

- **`agent/stories/newsworthiness.js`** — a **pure** `scoreNewsworthiness(item, { person, matter })`
  → `{ tags: [{kind, detail?}], score }`. Explainable tags: `money` (title `$`/`million`/`bond(ing)`/
  `TIF`/`appropriation`), `accountability` (police/surveillance/ethics/appointment/no-bid),
  `equity` (demolition/eviction/closure + a district), `novelty` (ordinance *creating*/pilot/
  first-of-kind), `anomaly` (reuse MOO-51 `walkOnFlag`/`consentFlag`), `recurrence` (repeat
  entity — optional v1, needs ownership portfolio MOO-110). Composite, transparent ranking.
  Reuse the title-signal style already in `salience.js` `bigReason`.
- **Story angle** — `agent/stories/angle.js`: a grounded one-line hook + "why it's a story",
  generated by Claude from the real matter (title/text/sponsor) via a schema (e.g.
  `STORY_ANGLE_SCHEMA = { hook, whyStory }`). **Reuse the summarizer plumbing**:
  `createClaudeGenerate({ schema })` from `agent/summarizer/index.js` (same pattern as
  `summarizeMatterBilingual`). Anti-fabrication: the prompt must say "use only the provided
  record; if unsure, say so." Bilingual where the channel is ES.
- **Surface A — App Home "📰 Story leads this week"** — a new section in
  `agent/blockkit/home-view.js`, shown **only for the reporter persona**
  (`channels.some(c => c.role === 'reporter')`). Reuse the MOO-123 feed/render pattern +
  the `discover_watch` button (see below).
- **Surface B — `/gavel stories [committee|topic]`** — extend `agent/listeners/commands/gavel.js`
  (and its registration) with a `stories` subcommand → ranked leads for a committee or a
  MOO-121 topic key.
- Each lead links the reporting starting points already in Gavel: sponsor + contact (MOO-72),
  🕓 history (reuse `alert_history`), transcript search (MOO-113) when available.

**Verification:** unit (newsworthiness tagging on money/anomaly/equity fixtures; deterministic
explainable ranking; angle schema-validated + cites the item; reporter-persona gating; empty
state) + live (reporter App Home shows real leads incl. a real walk-on/consent anomaly;
`/gavel stories licenses` returns ranked leads; **spot-check 3 angles against the real record
for accuracy — no fabrication**). Screenshot.

## The one real design decision (worth a quick think) — LLM latency

The story **angle** is a Claude call. **Do not generate angles synchronously while rendering
the App Home** (the Home must render fast; same reason MOO-122's sample card skips the LLM).
**Recommended split:**
- **App Home "📰 Story leads"** = newsworthiness **tags + title only** (LLM-free, fast,
  explainable). The "why it's a story" here is derived from the tags.
- **`/gavel stories`** = afford the latency: `ack()`, post a "🔎 Digging through this week's
  agenda…" status, generate angles for the top ~5 leads, then post. (Look at how the
  Ask-Gavel thread shows a thinking status for the pattern.)
- No new persistence (mirror MOO-123/122). If you later want angles in the Home, cache them
  on the detected row — but that's out of scope for v1.

## Gotchas you'd otherwise re-derive

- **Detected rows are thin** (`detectedAgendaItems`): only `eventItemId, eventId, matterId,
  title, eventBodyName, eventDate, agendaNumber, walkOnFlag, consentFlag`. **No** sponsor, `$`,
  fileNumber, or district. Sponsor/matter text come from `enrichForAlert(row, legistar)`;
  district is parsed from the **title** (`(Nth Aldermanic District)` — see `salience.js districtOf`).
- **`listUpcoming` returns ~100 rows** via the Convex client (page cap). Fine for a week's
  view; filter by committee/topic for `/gavel stories`.
- **Don't LLM-in-render** (see the design decision above). Enrich + angle only for the top N,
  and not on the App Home synchronous path.
- **Reporter gating:** `subscription.role` is on the row (`'association'|'organizer'|'reporter'`,
  MOO-117/118). Gate Surface A on a reporter channel existing.
- **App Home Watch button needs a channel** — unlike an alert card, the Home has no
  `body.channel.id`. Reuse the `discover_watch` pattern (opens the pre-filled add-watch modal);
  add a distinct `action_id` if you need a separate handler.
- **Slack `action_id`s must be UNIQUE within a view** (this bit FD-B + the topic chips). Give
  story buttons their own ids; suffix per-item if needed.
- **Bilingual:** committee names / file numbers / addresses stay English even in ES copy
  (project rule). Translate the hook/labels, not the civic identifiers.
- **biome:** one **pre-existing** error in the untouched `tests/alerts/match.test.js` — ignore it.

## Worktree / env setup (a fresh worktree needs this)

`.slack/`, `.env`, `.env.local`, `node_modules`, `convex/_generated` are **gitignored** and live
only in the main checkout `agent/`. For a new worktree off `main`:
1. `git worktree add .claude/worktrees/moo-127-story-radar -b <linear-branch> main` (MOO-127 is **independent — branch off `main`**).
2. `cd agent`, `npm ci`, copy `.env` + `.env.local` from the main `agent/`, then **append `CONVEX_URL` to the worktree `.env`** (`app.js` loads `.env` only, not `.env.local`).
3. `npx convex dev --once` to generate `convex/_generated` (deployment `vivid-weasel-903`).
4. **npm ci gotcha:** if agent-SDK-dependent tests fail with `ERR_MODULE_NOT_FOUND
   @anthropic-ai/claude-agent-sdk`, re-run `npm ci` (the first run can truncate). Confirm
   `node_modules/@anthropic-ai/claude-agent-sdk` exists.
5. **Only if running `slack run` locally:** copy `.slack/` from main `agent/`; and **pause Fly
   first** (`fly machine stop e8202d9a7d1078 -a gavel-app`) — the deployed app shares the same
   Socket Mode app (`A0B8GP68PLJ`), so a local run and prod fight over events. Restart after
   (`fly machine start e8202d9a7d1078 -a gavel-app`). `/gavel stories` needs **no manifest change**.
