### What we shipped

- **📬 "From the city" digest.** Pure aggregator (`civicmail/aggregate.js`): clusters the firehose by
  category, **folds routine into counts by record type** (the city buries each complaint after a colon
  in `description` — `DNS Activity:Large pile of tires` — so folding on the raw text never groups; fold
  on the prefix), classifies `other`/`newsletter` into **civic-life kinds** (press releases, community
  events, hearings, RFPs, jobs), flags **recurring license applicants**, dedups Legistar overlaps, with
  an optional district gate + `[since,until]` window. One batch bilingual Claude briefing per channel
  (not per email). A "From the city" Block Kit card with per-highlight 📖 Read buttons and the protected
  "How to be heard" footer. Auto-cadence via a `digestedAt` field (orthogonal to `alertStatus`),
  `--all`/`--mark` cron path, and a Tue/Fri crontab entry committed **disabled**.

- **🔎 Federated `/gavel search`.** Evolved across commits: back-catalog search → precision (quotes =
  exact phrase, multi-word = AND, fixing Convex's OR-noise) → **attachment text** (PDF agendas/packets
  indexed via Claude document blocks) → **OCR** (flyer JPGs read via Claude vision) → **hybrid semantic**
  (OpenAI embeddings + the `by_embedding` vector index, all 135 rows embedded) → **federation** across
  four sources grouped in the card: 📬 mail (keyword + semantic) · 🏛️ Legistar agendas (new `search_title`
  index) · 🎙️ minutes (vector) · 📖 zoning (vector).

- **🪟 Civic-record modal.** A 📖 Read button on every highlight + search result opens an in-Slack modal:
  email body + extracted PDF text + **flyers rendered inline as images** (fresh presigned AgentMail URL,
  no new Slack scope) + How-to-be-heard + Watch. Opens in <3s (no Claude on the open path). Replaces the
  useless generic E-Notify link.

### Decisions

- **Group federated results by source, not one cross-source ranking.** Keyword BM25 and cosine across
  different vector spaces aren't comparable — grouping sidesteps the false precision and reads clearer.
- **Quotes = exact, keyword-only; unquoted = hybrid.** Quotes are the escape hatch from semantic fuzz
  (which always returns *something*). A multi-word unquoted query requires all words (mail) but trusts
  Convex's title ranking for short agenda titles.
- **OCR + PDF text both via Claude (vision / document blocks), no `tesseract`/`pdf-parse` dependency** —
  matches the codebase's existing no-extra-deps pattern.
- **The Tue/Fri cron stays disabled** until live ingestion resumes and the demo is recorded: the source
  is a dormant snapshot, so `--mark` would post once then go quiet (consuming the demo data).

### Learnings / gotchas retired

- **The AgentMail inbox holds 135 messages, not 100** — the backfill was silently capped (paged with the
  wrong param; the advancing param is `page_token`, not `last_key`). Earlier "600" was a broken-pagination
  artifact.
- **The "3 raze orders" demo narrative isn't in the data** — 0 raze mentions this sample week. The honest
  aggregate pattern is recurring applicants (Foley & Lardner ×3). Do not fabricate raze orders.
- **`createClaudeGenerate` hard-capped `max_tokens` at 1024** — long agendas truncated mid-JSON. Added an
  optional `maxTokens` (default unchanged); attachment extraction bounds its own output so big packets
  don't overflow.
- **Two self-inflicted honesty bugs caught and fixed:** the digest card first pointed at `/gavel search`
  before it existed (→ `/gavel watch`), then the live cards referenced an undeployed command (deferred the
  re-post until after deploy). Verify "wired AND deployed," not just "written."
- **Convex full-text search is OR + prefix, no phrase operator** — `"data center"` matched anything with
  "center". Hence the precision layer.

### Risks retired

  searchable (federated search) **and** readable (modal) — the AgentMail beat is demo-ready.
- `/gavel search` is no longer mail-only; it spans the three-memory model's structured + semantic stores.
