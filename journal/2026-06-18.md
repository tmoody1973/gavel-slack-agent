### Decisions worth remembering

- **`permits` is a keyword, not a committee** (FD-A). Spec ¬ß3 listed it in the organizer *committees*
  column, but it isn't a Legistar `EventBodyName` ‚Äî in `committees` it would be dead data that never
  matches `match.js`. Emitted as `keywords: ['permit']`; the engine output gained a `keywords[]` field
  mirroring the subscriptions store.
- **Reporter "all committees" ‚Üí curated `CORE_COMMITTEES`** ‚Äî `match.js` has no wildcard and Milwaukee
  has ~169 bodies; "all" = the core Gavel actively covers, using only committee names verified in the
  codebase.
- **Reused the rich MOO-74 `homeView` as the configured hub** (added a "Set up another channel"
  button) and dropped the planned `homeConfigured` builder ‚Äî one hub, no dead code.
- **Route first-run on `channels.length`, not `configuredCount`** ‚Äî the demo workspace has
  poller-written subscriptions with `configured` undefined; branching on `configuredCount` would have
  regressed those existing users to a first-run screen. Surfaced + tested.
- **Lean nudge scope** ‚Äî first-`/gavel` + App-Home first-run fallback; the `member_joined_channel`
  bot-add trigger lives in FD-C's manifest change, not FD-B.
