### Learnings / gotchas retired

- **App-issued user tokens post with a `bot_id`** (`bot_profile: "Gavel (local)"`), so a
  synthetic "user" DM sent via `SLACK_USER_TOKEN` is skipped by our own bot-message guard.
  E2E thread tests need a real human message; the API can only *read* the result.
- **The 7-day question (from Tarik's live test):** `get_upcoming_events` caps at 7 days +
  Final-only by design, but the deeper truth is Milwaukee's Legistar **Web API** only
  carries events near agenda publication — a full no-filter query through July 14 returned
  9 events, furthest June 15, all Final. The legistar.com calendar pages (InSite) show
  scheduled dates the API doesn't have yet. Wider windows buy nothing; honest tool
  description wording is the right fix (small follow-up candidate).
- Per-call `createSdkMcpServer` construction is cheap (no transport until `query()` wires
  it; SDK schema registry is a WeakMap) — verified during review, no caching needed.
