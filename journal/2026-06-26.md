## Session 3 (same day) — Civic News Enrichment (MOO-179) built, merged, deployed

Took the approved civic-news spec from `writing-plans` all the way to deployed, in one subagent-driven run.
`main` moved to `effd16c`; **910 tests green** (879 → +31). Both Fly apps redeployed.

### What we shipped (MOO-179, PR #53 → `effd16c`)
A new `agent/news/` module connecting **external local reporting to the specific government item Gavel
tracks**, on two surfaces: a `📰 In the local news` block on alert cards, and a 5th `📰 Local news` lane in
`/gavel search`. Google News RSS behind a `NewsSource` interface (hand-rolled parser, no new dep; Exa adapter
= documented seam) → Claude relevance gate (`NEWS_GATE_SCHEMA`) → read-through Convex `newsCache` (24h TTL,
keyed by file # for alerts / normalized query for search). Every surface degrades to no-news on any failure.

### How we built it
- `writing-plans` → a 10-task TDD plan with complete code per task (zero placeholders), anchored to real
  integration points via an Explore pass first.
- `subagent-driven-development`: fresh implementer per task + a spec/quality reviewer after each, in an
  isolated worktree (native `EnterWorktree`). Controller (me) curated each brief as a file, kept a durable
  ledger, never pasted history between dispatches.
- **The review loop earned its keep** — it caught two real defects before merge:
  1. Task 7: `.slice()` on un-guarded awaited results + an unnormalized search query → explicit null-guards +
     normalized query + degrade tests.
  2. Final whole-branch review (opus): the **Convex `null` vs `v.optional`** bug (below).
- A **live smoke test** (real Google News RSS + real Claude gate + real Convex write/read-back) for the
  Midtown data center returned exactly the on-topic *Milwaukee Journal Sentinel* article and the cache write
  landed — exercising the real RSS/Claude/Convex path the unit suite stubs.

### Learnings / gotchas retired
- **Convex `v.optional(v.string())` rejects `null`** (accepts only `string | undefined`). The RSS parser
  emitted `source: null` for missing tags, so `upsertCache` silently rejected those rows — and because the
  write was `.catch(()=>{})`-guarded, the cache just never populated (re-fetch + re-gate every time, invisibly).
  Unit tests can't catch it (Convex validation runs in the Convex runtime, not `node --test`); only the live
  smoke test did. Fix: strip nullish optional fields before caching. Saved as memory `convex-optional-not-null`.
- **Fresh worktree lacks `convex/_generated`** (gitignored) → baseline tests fail at load until it's copied
  from main; and codegen must run in the *main* repo before deploy (the worktree's `_generated` is gone) or the
  image crash-loops on `api.newsCache`. Same root cause as the existing `convex-codegen-before-deploy` memory —
  held again.
- **Worktree commit discipline** — every subagent bash command chained an absolute `cd` into the worktree +
  a post-commit `git branch --show-current` check, per `subagent-cwd-leaks-to-main`. No leaks this run.
