## Decisions

- **Demo, not build, is the remaining lever.** The agent is over-built for a 3-min window; said so plainly and
  kept steering toward the recording + submission.
- **AgentMail send goes through the REST API, not the SDK.** The `agentmail` package isn't in the app's
  runtime node_modules, so the SDK import would crash boot. Matched the app's existing fetch+Bearer pattern.
- **News tool: connect, don't feed.** Decided the value is tying outside reporting to a specific government
  item (query + Claude relevance gate), surfaced on the alert + as a 5th search lane — not a generic digest.
  Google News RSS now behind a `NewsSource` interface, Exa adapter later. Slotted after the submission.

## Learnings / gotchas retired

- **Two Fly apps, redeploy both.** The "✍️ button is missing on alerts" report had a clean root cause: I'd
  redeployed `gavel-app` (handles the click) but not `gavel-poller` (posts the cards via `poll-once.mjs`).
  The poller was running pre-button code. Redeployed it (v10); the button is now in the alert pipeline. Any
  change to a shared card builder needs both apps redeployed.
- **The `agentmail` SDK isn't installed in the app** — caught only because I verified the send live (HTTP 200)
  instead of trusting the code. Classic case for live verification.
- **getItem live-resolves the title.** Wiring `getItem` to Legistar matter-by-file (MatterTitle/MatterBodyName)
  made the comment modal show the real item title, confirmed in the delivered email.
- **Codegen-before-deploy held again.** `_generated` is gitignored but image-baked from the build context, so
  `npx convex dev --once` in the deploy directory is required before shipping a new table (civicComments).
