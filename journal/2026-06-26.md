## Decisions

- **Demo, not build, is the remaining lever.** The agent is over-built for a 3-min window; said so plainly and
  kept steering toward the recording + submission.
- **AgentMail send goes through the REST API, not the SDK.** The `agentmail` package isn't in the app's
  runtime node_modules, so the SDK import would crash boot. Matched the app's existing fetch+Bearer pattern.
- **News tool: connect, don't feed.** Decided the value is tying outside reporting to a specific government
  item (query + Claude relevance gate), surfaced on the alert + as a 5th search lane — not a generic digest.
  Google News RSS now behind a `NewsSource` interface, Exa adapter later. Slotted after the submission.

## Learnings / gotchas retired

- **Two Fly apps, redeploy both.** The "✍️ button is missing on alerts" report had a clean root cause: I'd
  redeployed `gavel-app` (handles the click) but not `gavel-poller` (posts the cards via `poll-once.mjs`).
  The poller was running pre-button code. Redeployed it (v10); the button is now in the alert pipeline. Any
  change to a shared card builder needs both apps redeployed.
- **The `agentmail` SDK isn't installed in the app** — caught only because I verified the send live (HTTP 200)
  instead of trusting the code. Classic case for live verification.
- **getItem live-resolves the title.** Wiring `getItem` to Legistar matter-by-file (MatterTitle/MatterBodyName)
  made the comment modal show the real item title, confirmed in the delivered email.
- **Codegen-before-deploy held again.** `_generated` is gitignored but image-baked from the build context, so
  `npx convex dev --once` in the deploy directory is required before shipping a new table (civicComments).

### What we shipped
- **MOO-171 drafting-placeholder fix (PR #52 → `a5bcfb5`, deployed `gavel-app` v48).** The comment modal
  opened with an *editable, submittable* template in `civic_comment_body`; the Claude draft only swapped in a
  beat later via `views.update`, so a fast submit filed the bare template (the live bug from session 1). Fix:
  `buildCommentModal` gains a `drafting` flag — while true the editable input is replaced by a **non-editable**
  "✨ Gavel is drafting…" placeholder (EN + ES), so there's nothing submittable until the real draft swaps in.
  Open handler opens in drafting state when a Claude boundary is wired, then `views.update`s the editable draft;
  **falls back to the editable template if the draft fails** so the user is never stranded. Submit handler
  unchanged — the existing empty-body guardrail safe-degrades a submit-during-drafting.
- Built TDD: 6 tests RED-first (modal drafting EN/ES; open drafting / swap / fallback / no-boundary), verified
  4 failed for the right reason, then GREEN. **Live-verified in Slack by Tarik** — placeholder → editable draft
  swap, bare template not submittable. Linear MOO-171 → Done with an evidence comment.

### Gotchas retired
- **`gh pr merge` succeeded server-side but couldn't fast-forward local `main`** — because local `main` had
  three commits that were never pushed (the civic-news spec, the handoff, today's journal). Branching the fix
  off local `main` and pushing carried those three up, so GitHub's squash folded **all of them + the fix** into
  one commit (`a5bcfb5`). Verified the squash contained every file, then reconciled local `main` to the remote,
  **stashing/restoring the pre-existing uncommitted doc edits so nothing was lost.** Lesson: keep local `main`
  pushed, or a squash-merge will silently flatten your unpushed commits into the PR.
