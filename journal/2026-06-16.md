### Surprises / learnings

- **The wired permits dataset (`828e9630‚Ä¶`) has NO owner column** ‚Äî only Address (+ type/
  status/cost/use/dwelling-units). That's the *reason* LLC‚Üípermit needs the MPROP ownership
  join; an LLC can't be matched against the permit rows directly. Confirmed by a live
  one-row CKAN query before designing.
- **The data reference (line 103-107) explicitly blesses the watchlist sweep shape** ‚Äî
  `/matters?$filter=substringof(...,MatterTitle)&$orderby=MatterIntroDate desc` ‚Äî so the
  "query new matters citywide" approach beat "reuse the agenda stream" cleanly.
- **Convex validator vs injected clock:** first verify run failed because `runWatchSweep`
  passes `alertedAt` but the mutation validator only had the 4-field key + re-stamped
  server-side. Fixed by widening the insert validator (`alertRecord`) to store the
  caller's `alertedAt` ‚Äî keeps the unit test's deterministic clock honest.
- **Verify-script term derivation bug:** first attempt picked two non-adjacent title words
  ("Presentation discussion" from "Presentation **and** discussion‚Ä¶") ‚Üí no substring match.
  Switched to a single distinctive word. Good reminder that the *test harness* can be wrong
  while the unit under test is right.
