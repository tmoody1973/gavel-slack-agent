## Decisions made
1. **Captions over Deepgram** for transcripts (free, timestamped, daily) — Deepgram is fallback only.
2. **CKAN over LIRA** for permits/licenses; **no Firecrawl** (structured JSON API beats scraping a JS form).
3. **County is in scope** as a client-flip (legislation-only; no parcel/permit for county).
4. **Milwaukee Turners** as first design partner + likely the real "Potential Impact" story for the submission.
5. **Scope freeze** (end-of-session call): MOO-66/67/68 are parked; the next work is the spine.

## What I learned (Tarik)
- **The public website ≠ the API** — a UI default filter made county look empty; the data was all there.
- **Search by *current* terminology** — "Community Reintegration Center," not "House of Correction"; the old keyword only found 2011–2018 matters.
- **Structured API > scraping** — reach for Firecrawl/headless only when a target is JS-rendered and has no API.
- **The solo-hackathon trap:** validating and scoping feels productive, but the demo is won by *one working workflow*, not breadth.
