## What I learned (Tarik)
- **The public website ≠ the API** — a UI default filter made county look empty; the data was all there.

- **Structured API > scraping** — reach for Firecrawl/headless only when a target is JS-rendered and has no API.
- **The solo-hackathon trap:** validating and scoping feels productive, but the demo is won by *one working workflow*, not breadth.

## TL;DR

Built the product's brain and the routing backbone. **MOO-42 (Claude summarizer)** is merged to `main`; **MOO-45 (Convex subscription state)** is in review. Both follow one testable pattern: pure logic unit-tested deterministically, the non-deterministic / infra boundary proven against *real* data. Suite at 42/42. Convex dev deployment is live.

## What we did

### Clean base first
- Merged **PR #1** (the moo-38 agent scaffold) → `main` and deleted the stale branch. Ended the "docs on main / code on a branch" split. `main` is now the single base; one branch per issue off it.

### MOO-42 — Claude summarizer ✅ Done (PR #2 merged)
- Real Legistar matter → plain English (≤80 words) + a one-line "why it matters" + extracted street addresses.
- **Design seam (load-bearing):** `summarizeMatter(matter, { generate })` — the Claude call is *injected*, so the logic (source fallback chain, prompt assembly, output validation, word count) is unit-tested without a network; the non-deterministic call lives in `client.js` and is proven against real matters. This pattern propagates to every Claude-touching module in the spine.
- **Stack:** `@anthropic-ai/sdk`, **Sonnet 4.6** (per PRD + the issue), structured output via `output_config.format`.
- Verified on 4 real matters (252190 the 234 S Water St rezone, 26067 appeal @ 10835 W Mill Rd, 260243 commendatory communication, R13777 minutes approval): summaries 37–75w, addresses correct vs. source, sparse procedural item summarized with **no hallucination**.

### MOO-45 — Convex subscription state 🔍 In Review (PR #3)
- `subscriptions` table: one row per channel — committees, keywords, optional district boundary, **per-channel `language`** (EN default / ES). Indexed `by_channel` / `by_client`. Client-aware (`milwaukee` | `milwaukeecounty`).
- **Minimal PII by construction:** `normalizeSubscription` whitelists fields; Convex arg validators reject unknown fields. No Slack user IDs, no message content — the ToS guardrail enforced in code.
- CRUD: `upsert / get / list / setLanguage / remove`.
- Stood up **Convex** from scratch (dev deployment `vivid-weasel-903`); `convex/` in TypeScript (typed codegen), agent stays JS.
- Verified live: junk fields (`userId`, `messageText`) rejected at the validator boundary; stored row leaked none; a real row drove target + language and flipped EN→ES; CRUD + cleanup confirmed.

## Decisions made
1. **Merge-to-main base** (not stacked branches) — PR #1 merged; branch-per-issue off `main`.
2. **Injected-client test seam** for every Claude/infra module — unit-test the logic, verify the boundary against reality (Tarik's call when asked Socratically).
3. **Sonnet 4.6** for summarization (cost/latency — the poller runs it per item).
4. **`convex/` in TypeScript, agent stays JS** — typed codegen where it pays; no whole-app TS migration (scope).
5. **PR-first review** for both load-bearing issues. CodeRabbit is on Free (summary/walkthrough only, no line-by-line) — did a manual critical pass instead.

## What I learned (Tarik)
- **Dependency injection as a test seam:** the way to TDD a non-deterministic LLM call is to *inject* it — unit-test the deterministic logic around it, prove output quality against real data separately. The load-bearing pattern for the whole spine.
- **Milwaukee `MatterText` is empty** across the full MatterId range swept — the title carries the substance, which is *why* addresses are extracted from titles (reconfirms MOO-37).
- **Convex serializable OCC** makes check-then-insert upserts race-safe without a unique constraint.
