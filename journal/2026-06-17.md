### Decisions

- **Sub-task D storage = a new `matterOutcomes` table** (not folding onto `detectedAgendaItems`).
  Matches the PRD's "vote-record compilation" intent and captures every acted-on item, not only the
  ones we alerted on pre-vote. Detection (pre-vote) and outcome (post-vote) are different temporal
  phases, so they're different tables.
- **`get_video_moment` takes `eventId` as well as `eventItemId`** ‚Äî forced by the Legistar API

- **Prepped the manifest + had the human reinstall** rather than touching live Slack config silently
  ‚Äî a scope change + reinstall is an owner action.

### Learnings / hard-won facts

- **Legistar has no top-level `/eventitems/{id}` (404).** An item is exposed only within its meeting:
  `/events/{eventId}/eventitems/{itemId}`. There's also no global `/eventitems` OData collection.
  Caught live when the first `get_video_moment` wiring 404'd against an HTML error page.
- **A scope-only Slack reinstall did NOT change the bot token value** (token rotation is off), so no
  `.env`/Fly-secret update was needed ‚Äî the existing `xoxb` just gained `files:write`. Worth knowing
  before assuming a reinstall means re-plumbing every secret.
- **Milwaukee committee votes are voice votes** ‚Äî `EventItemSeconder` / `EventItemTally` come back
  null; the real outcome signal is `EventItemActionName` + `EventItemPassedFlagName` + `EventItemMover`.
  The parser drops null optionals and collapses the double-space Legistar returns in
  `"RECOMMENDED FOR  ADOPTION"`.
- **`node --test <dir>` under-reports** (showed `pass 0 / fail 1` mid-session); bare `node --test`
  gave the true 363/0. Re-confirmed the existing gotcha.
- **Shared Convex deploy was safe because the branch schema is a superset of main** ‚Äî `npx convex dev
  --once` reported *only* the three new `matterOutcomes` indexes added, nothing removed. Verified the
  table list before deploying, per the last-writer-wins hazard.
